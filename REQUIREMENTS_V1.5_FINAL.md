## 二、技术选型（已定）

**最优架构**：

```
faster-whisper medium（本地）
+ 句子缓冲区
+ WebSocket 保活
+ PM2 守护进程
```

**不采用**：Deepgram / Google 云端 ASR
理由：成本高、依赖网络、隐私风险、合规风险，不适合长期稳定产品。

## 三、需求详细（P0 + P1 全部必须满足）

### P0：稳定性修复（必须完成）

#### P0-1：连续运行 2 小时以上【最高优先级】

**现状**：运行约 5 分钟后出现各类崩溃/停止

**技术方案**：
- WebSocket 心跳重连（30s 间隔，断线 5s 内自动重连）
- 音频流看门狗（30s 无活动 → 自动告警 + 恢复）
- PM2 进程守护（崩溃自动重启，重启间隔 ≤ 3s）
- `/tmp` 残留音频文件定期清理（每小时）

**验收标准**：
- [ ] 连续录音 2 小时，服务不崩溃、不卡死
- [ ] 网络短暂断线（< 30s）后自动恢复，无需手动干预
- [ ] PM2 监控面板可查看进程状态和重启次数
- [ ] 内存使用平稳，无明显泄漏

#### P0-2：修复 Bug C — HTTP 530 崩溃

**现状**：Cloudflare Tunnel 偶发 530，导致服务整体崩溃

**技术方案**：
- PM2 进程守护 + 崩溃自动重启
- 全局异常捕获（uncaughtException + unhandledRejection）
- 崩溃日志持久化（通过已有 V1.4 日志系统）

**验收标准**：
- [ ] HTTP 530 发生后，服务 3s 内自动恢复
- [ ] 崩溃原因记录到日志，可追溯
- [ ] 用户端看到连接重建，而非白屏

#### P0-3：修复 Bug B — 来电中断后恢复

**现状**：来电期间录音中断，挂断后无法自动恢复

**技术方案**：
- iOS AudioSession 中断事件监听
- 中断结束 → 5s 内自动重新 prepare + resume 录音
- UI 明确提示"录音已暂停 - 来电"/ "正在恢复..."

**验收标准**：
- [ ] 来电接听 + 挂断后，5s 内录音自动恢复
- [ ] UI 有明确的中断状态提示
- [ ] 恢复后识别结果连续（不丢失中断前内容）

### P1：体验优化（必须完成）

#### P1-1：升级 ASR 模型到 faster-whisper medium

**现状**：使用 whisper-base，准确率约 80-85%

**技术方案**：
- 将 `WHISPER_MODEL=base` 替换为 `faster-whisper medium`
- 使用 faster-whisper（比 openai-whisper 快 4x，内存更少）
- Worker 数量：2-3（根据机器内存调整）

**性能基准**：
- 准确率：97%（广播级）
- 延迟：2-3s（满足标准）
- 内存：约 1.5GB（medium 模型）

**验收标准**：
- [ ] 用马斯克演讲音频（已有标准测试素材）测试，准确率 ≥ 97%
- [ ] ASR 延迟 ≤ 3s（90th percentile）
- [ ] 服务启动后 30s 内 Whisper 模型加载完成

#### P1-2：句子缓冲区 — 解决翻译跳动

**现状**：1s 碎分片 → 每片都翻译 → 翻译频繁跳动，体验差

**技术方案（行业最优解）**：
- ASR 流式转写：英文实时逐字追加（用户能实时看到）
- 句子边界检测：
  - 方式1：Whisper 输出的标点符号（句号/问号/感叹号）
  - 方式2：静音停顿（> 800ms）
- 句子缓冲区：满足边界条件 → 整句送翻译 → 中文整句输出
- UI 策略：英文区实时滚动 / 中文区整句追加（不跳动）

**验收标准**：
- [ ] 英文区：识别结果实时追加，延迟 ≤ 1s
- [ ] 中文区：整句输出，不出现"翻一半"或频繁更新
- [ ] 连续说 2 分钟，中文翻译无明显跳动

## 四、会话历史与导出功能

### 4.1 历史会话详情

- **去掉生词表**：历史会话详情页不展示生词列表（保持简洁）
- 只展示英文字幕和中文翻译的对应内容

### 4.2 导出按钮（新增 P1）

**功能**：在会话详情页增加"导出"按钮，一键导出为带时间戳的 Markdown 文件

**导出格式**：

```markdown
# voice-bridge 录音记录
**录音时间**：2026-03-28 14:30:00
**时长**：1小时 23分钟

---

[00:00:05] **EN**: And for most of actual human history, China has been the most powerful nation on earth.
[00:00:05] **CN**: 在人类历史的绝大部分时间里，中国一直是地球上最强大的国家。

[00:00:12] **EN**: And so you can expect that they will do many great things.
[00:00:12] **CN**: 因此，你可以期待他们会做出许多伟大的事情。
```

**用途**：
1. 用户方便保存、分享内容
2. 出现 Bug 时，用户导出内容直接发给团队，快速定位问题

**验收标准**：
- [ ] 会话详情页有"导出 Markdown"按钮，位置显眼
- [ ] 每句英文/中文均带时间戳（格式：`[HH:MM:SS]`）
- [ ] 导出文件名：`voice-bridge-YYYY-MM-DD-HH-MM.md`
- [ ] 导出文件可通过 iOS 系统分享（Share Sheet），可存入 Files app
