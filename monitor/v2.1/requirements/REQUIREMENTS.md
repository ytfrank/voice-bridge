## 一、背景

### 1.1 为什么做安卓版

- V2.0 已完成 iOS 端开发，核心链路已验证（Deepgram ASR + GLM-4-flash 翻译）
- 波哥需要在**小米手机**上使用 Voice Bridge
- 最终目标：推送到**安卓应用市场**（Google Play / 小米应用商店）

### 1.2 V2.0 已验证能力（安卓版基线）

| 模块 | V2.0现状 |
|------|---------|
| ASR引擎 | Deepgram Nova-3（REST API），延迟 0.9~1.6s |
| 翻译引擎 | 智谱 GLM-4-flash（不动） |
| BFF后端 | Node.js Express，端口3001，公网可访问 |
| 质量门控 | 静音拦截 + 幻觉拦截 + 文本质量评估 |
| 前端框架 | React Native + Expo SDK 54 |
| app.json | 已配置 android 字段（package: com.voicebridge.app） |

### 1.3 技术优势：Expo 跨平台

项目基于 Expo SDK 54 + React Native，**天然支持安卓**。app.json 已有安卓配置。核心工作量在于：
- 安卓特有的音频权限处理
- 安卓模拟器/真机调试
- APK/AAB 构建和签名
- 应用市场提审

## 四、技术方案（Peter确认）

### 4.1 前端：Expo 跨平台（改动量小）

React Native + Expo 已天然支持安卓。主要工作：
1. **EAS Build 配置**：eas.json 添加 android build profile
2. **音频模块适配**：expo-audio 在安卓上的权限和录音行为可能有差异
3. **UI适配**：安卓 SafeArea / 状态栏 / 导航栏可能需要微调

### 4.2 后端：不变

BFF (Node.js) 是独立服务，不区分 iOS/Android。已通过 Deepgram Nova-3 验证。

### 4.3 部署方案（Atlas确认）

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Cloudflare Tunnel**（快速） | 0配置，5分钟搞定 | 临时URL，不稳定 |
| **voice.doramax.cn + VPS**（正式） | 正式域名，SLA保证 | 需要VPS部署BFF |

**建议**：明天先用 Cloudflare Tunnel 完成小米测试，P1阶段切到正式域名。

### 4.4 安卓构建方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **EAS Build（云端）** | Expo官方，零配置 | 免费额度30次/月 |
| **本地 Android Studio** | 无限制，调试方便 | 需要配置 Android SDK |

**建议**：Peter评估选择。
