# TECH_PLAN.md — voice-bridge V1.7 Phase 1

## 目标

在 **不换模型**（继续 `Whisper medium + GLM-4-flash`）前提下，完成 Phase 1：

1. 修复空结果问题
2. 降低幻觉问题
3. 优化端到端延迟
4. 支持服务按需启停
5. 维持目标场景：3人以内，广播 / 面对面交流

## 范围

### P0
- iOS 录音 / 上传链路问题排查与修复
- ASR 低质量结果拦截（低置信度 / no_speech / 异常文本）
- 空文本 / 低质量文本不进入翻译

### P1
- chunk 策略优化（2~3s + overlap / 端点策略）
- worker queue 优化（session 串行、全局限流、超时淘汰）
- start/stop 脚本（默认关闭，需要时启动）

## 预期指标

- 空结果率：84.4% → 5~10%
- 幻觉率：<5%
- 端到端延迟：12s → 5~7s（保守 7~9s）

## 技术判断

- 当前主要瓶颈不是单纯模型精度，而是录音链路 + chunk 边界 + worker 排队 + 缺少拦截层。
- Phase 1 优先通过链路修复与调度优化拉回可用性，暂不切模型。
- 广播场景以稳定和低空结果率优先，准确率提升以“减少错误放大”为主要手段。
